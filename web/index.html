graphics

Native all-C graphics cmod for MicroPython, CircuitPython, and CPython. import graphics loads a single C module — no Python package inside this repo. It provides Area, a framebuf-compatible FrameBuffer, format constants (RGB565, RGB888, mono/GS variants, …), plus framebuf_backend(), capabilities(), and implementation().

pydisplay keeps a pure-Python src/lib/graphics fallback. When this cmod is linked or installed, graphics.framebuf_backend() reports "native" and implementation() reports native_cmod.

API excerpt

From the in-repo smoke tests 
from graphics import Area, FrameBuffer, RGB565, framebuf_backend

assert framebuf_backend() == "native"

a = Area(0, 0, 10, 10)
assert a.contains(5, 5)
assert Area(0, 0, 10, 10).clip(Area(5, 5, 10, 10)) == Area(5, 5, 5, 5)

buf = bytearray(128 * 128 * 2)
fb = FrameBuffer(buf, 128, 128, RGB565)
fb.fill(0)
area = fb.fill_rect(10, 10, 20, 20, 0xF800)
assert area == Area(10, 10, 20, 20)

Build & test

CPython

Editable install, then run the test scripts.

python3 -m venv .venv
.venv/bin/pip install -e .
.venv/bin/python test_area.py
.venv/bin/python test_graphics.py
.venv/bin/python test_subclass.py

MicroPython

Clone as a sibling of micropython/. USER_C_MODULES is the workspace parent.

cd micropython/ports/unix
make submodules
make USER_C_MODULES=../../..

./micropython/ports/unix/build-standard/micropython \
  graphics/test_area.py
./micropython/ports/unix/build-standard/micropython \
  graphics/test_graphics.py
